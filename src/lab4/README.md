## Вариант 10. Медицина

### Класс:
* Patient - пациент
 
## Созданные интерфейсы

### 1. Printable
Метод `to_string()` - возвращает строковое представление объекта
`to_short_string()` - возвращает краткое представление объекта

### 2. Comparable  
Метод `compare_to(other)` - сравнивает объекты:
- `-1` если текущий меньше
- `0` если равны
- `1` если текущий больше

## Реализация в классах

| Класс | Printable | Comparable |
|-------|-----------|------------|
| Patient | выводит ФИО, ID, возраст, температуру, диагноз, статус | сравнивает по ID, ключ сортировки: ФИО + возраст |
| PatientRegistry | выводит список всех пациентов | фильтрует и сортирует объекты через интерфейсы |

## Выводы
1. Созданы два интерфейса - Printable и Comparable через абстрактные базовые классы (ABC)
2. Класс Patient реализует оба интерфейса:
- Printable - предоставляет 3 формата вывода и краткое представление
- Comparable - сравнивает пациентов по ID и предоставляет ключ сортировки
3. Класс PatientRegistry использует интерфейсы:
- Фильтрация объектов через get_printable_items() и get_comparable_items()
- Сортировка через sort_by_comparable()
- Вывод через print_all_via_interface()
4. Интерфейсы используются как типы в универсальных функциях:
- print_all_printable(items: list[Printable])
- compare_two_comparable(a: Comparable, b: Comparable)
5. Реализован полиморфизм - один интерфейс, разное поведение:
- to_string("short"), to_string("default"), to_string("detailed")
- Разные маркеры температуры у разных пациентов в to_short_string()
6. Достигнута архитектурная гибкость:
- Код зависит от интерфейсов, а не от конкретной реализации
- Новые классы могут легко добавиться, реализовав существующие интерфейсы
- Сортировка и вывод работают единообразно для всех объектов
